# Politique de securite

## Signaler une vulnerabilite

Ne publiez pas une vulnerabilite dans une issue publique. Utilisez la fonction
**Report a vulnerability** de l'onglet Security du depot GitHub.

Indiquez les versions concernees, l'impact, les etapes de reproduction et toute
mesure de mitigation connue. Ne transmettez jamais de token Klixa reel.

Les rapports seront accuses reception dans un delai cible de 5 jours ouvrables.
Les details pourront rester confidentiels jusqu'a la publication d'un correctif.

## Perimetre

Cette politique couvre le code de Klixa Companion et son service Raspberry Pi.
Le service cloud Klixa n'est pas distribue dans ce depot et dispose de son propre
processus de securite.
